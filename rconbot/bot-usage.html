Bot Usage

Moogly

The process by which the Mod Mail bot is used is pretty straight forward. Below I will illustrate the steps and flow for the bot and include screenshots the illustrate how it works from varying perspectives.

User Messages Bot

The bot works off of a 'Ticket' system, when a brand new user messages the Mod Mail bot a new ticket is created in the back-end. There is nothing the user has to do, this is to keep the barrier to entry very simple.

Â 

Sidenote: Tickets have titles that should be updated by any mod or anybody reviewing existing tickets in order to make tickets easier to sort through as they age.

So the first step in the bot-user flow is a user messaging the bot.

This will be sent to the bot, then the bot will forward the details to a mod only accessible channel.

This is step two in the process.

Then a mod can respond via the $msg command and the bot will show the response in the channel, this is part of the step three bit of the process which involves any back and forth between the user and the moderator through the bot.

Note that although we know which mod responded to the user, they do not know. This is probably to our advantage as it allows any mod to take over an exchange and an owner to be elevated if necessary into handling a user issue through the bot.

Here you can see the full exchange as it looks to the user thus far.

This can continue until a mod closes the ticket in the mod-mail channel by issuing the $close / $closeticket command.

Once the ticket is closed, mods cannot message a user through the bot until a user creates a new ticket. If a mod wishes to contact a user any further they should do it directly. This entire exchange is recorded to a database we fully control in order to give us total control of our data in the case where we may have to delete highly sensitive content that Discord themselves would not approve of being within our mod-mail channel.
